# Ontology Requirements (User Stories & Competency Questions)

This directory contains the functional requirements for the **AestheticsKG** ontology. These requirements are defined through personas, scenarios, Competency Questions (CQs) and SPARQL queries to ensure the ontology meets the needs of its diverse stakeholders.

## Structure
Each requirement file follows a standardized format:
- **Persona:** A description of the stakeholder (researcher, designer, developer, etc.).
- **Scenario:** The real-world context and motivation for using the ontology.
- **Competency Questions:** Specific questions the ontology must be able to answer.
- **SPARQL Implementation:** Practical queries that retrieve answers from the knowledge graph.

## Methodology
The requirements are used to validate the ontology's design. By running the provided SPARQL queries, we verify that the classes and properties correctly represent the domain of digital aesthetics and satisfy the users' needs.
